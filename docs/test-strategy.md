# Test Strategy (básica)

## Alcance 
- Login (UI)
- Validaciones de email/password
- Bloqueo por intentos fallidos (si aplica)

## Tipos de prueba
- Funcional: positivo/negativo
- Borde: Boundary Value Analysis
- Particiones: Equivalence Partitioning
- Smoke: validación rápida de flujo principal
- Regression: re-ejecución tras cambios

## Criterios de salida
- Casos críticos ejecutados (Smoke)
- Bugs S1/S2 reportados y re-evaluados
